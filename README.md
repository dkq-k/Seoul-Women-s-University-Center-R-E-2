# Seoul-Women-s-University-Center-R-E-2

## 1. 연구의 정의

연구(Research)는 아직 명확한 답이 없는 문제에 대해 체계적인 방법을 통해 답을 도출하는 과정이다.

단순한 지식 습득이 아니라, 새로운 지식을 만들어내는 과정이며 다음과 같은 태도가 요구된다.

- 근거 기반 사고
- 재현 가능성 확보
- 비판적 분석

---

## 2. 연구 진행 과정

연구는 다음과 같은 5단계로 진행된다.

1. 문제 정의  
2. 가설 설정  
3. 프로토타입 개발  
4. 검증  
5. 결과 공유  

이 과정은 단방향이 아니라 반복적인 구조를 가진다.

---

## 3. 연구 주제 선정 기준

연구 주제는 다음 기준을 만족해야 한다.

- Feasible: 현재 자원으로 수행 가능해야 한다  
- Interesting: 연구 가치와 흥미가 있어야 한다  
- Novel: 기존 연구 대비 새로운 요소가 있어야 한다  
- Ethical: 윤리적으로 문제가 없어야 한다  
- Relevant: 실제 문제 해결에 기여해야 한다  

---

## 4. 프로젝트 주제 정의

연구 주제는 다음과 같은 형식으로 정의한다.

우리는 [누구]를 위해  
[어떤 보안 문제]를 해결하는  
[어떤 결과물]을 만든다  

### 우리 팀 주제

우리는 리눅스 사용자를 위해  
커널 취약점을 체계적으로 탐색하기 어려운 문제를 해결하는  
CVE 기반 리눅스 커널 취약점 탐색 도구를 만든다  

---

## 7. 논문 읽기 방법

효율적인 논문 분석을 위해 3단계 접근법을 사용한다.

1차: 제목, 초록, 결론 중심 스캔  
2차: 방법과 결과 중심 이해  
3차: 알고리즘 및 수식 수준의 심층 분석  

---

## 8. 실험 설계

### 변수 구성

- 독립변수: 실험자가 조작하는 값  
- 종속변수: 결과로 측정되는 값  
- 통제변수: 일정하게 유지되는 값  

### 핵심 요소

- 재현성 확보 (random seed 고정)  
- 데이터셋 및 환경 기록  
- 공격자 모델 명확화  
- baseline 설정  

---

## 9. 정보보호 연구 윤리

### 허용

- 가상 환경 실험  
- 더미 데이터 사용  
- 방어 및 탐지 중심 연구  

### 금지

## 10. 논문 구조 (IMRaD)

- Introduction: 연구 필요성  
- Methods: 실험 방법  
- Results: 실험 결과  
- Discussion: 결과 해석 및 의미  

---

### 프로젝트명
리눅스 커널 취약점 후보 자동 발굴 도구 개발 연구

- CVE 기반 분석  

### 사용 기술

- C  
- Linux Kernel  
- LLM
